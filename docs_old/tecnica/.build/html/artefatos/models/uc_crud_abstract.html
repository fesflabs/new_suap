Objetivo¶

Este caso de uso descreve as funcionalidades báscias de Incluir, Excluir, Alterar, Listar e Visualizar uma entidade.

Via de regra todos os caso de uso do tipo CRUD irão especializar este caso de uso.

As referencias aos Requisito de Interface (RI), Requisito de Informação (RIN) serão descritos no caso de uso especialista.

Pré-condições¶

O ator precisa está autenticado no sistema.

Fluxo de Eventos¶

Fluxo Normal¶

  1. O caso de uso é iniciado acionando a opção “ ” RI1

  2. O sistema exibe a lista de entidades cadastradas conforme RIN1.

  3. O ator filtra (RIN1-3) a lista de entidades.

  4. O sistema exibe uma lista que satisfaça aos critérios informados no passo anterior.

Fluxo Alternativo¶

FA1 – Cadastrar Entidade¶

  1. Incluir passos 1 e 2 do Fluxo Normal.

  2. O ator seleciona a opção Cadastrar definido em RI2.

  3. O ator informa os dados definidos em RIN2 (RN1).

  4. O sistema exibe a mensagem M1.

  5. O caso de uso retorna para o passo 2 do Fluxo Normal.

FA2 - Alterar Entidade¶

  1. Incluir passos 1 e 2 do Fluxo Normal.

  2. O ator aciona a opção Editar (padrão definido no SUAP).

  3. O sistema exibe a entidade com os dados preenchido.

  4. O ator informa novos valores para os dados definidos em RIN2 (RN1).

  5. O sistema exibe a mensagem M2.

FA3 - Excluir Entidade¶

O padrão adotado pelo SUAP é não disponibilizar essa opção ao ator. Se for necessário disponibilizá-la, deve descrever os fluxos no caso de uso especialista.

FA4 – Visualizar entidade¶

O padrão adotado pelo SUAP é não disponibilizar essa opção ao ator. Se for necessário disponibilizá-la, deve descrever os fluxos no caso de uso especialista.

Fluxo de Exceção¶

Implícito na seção Regras de Negócio.
