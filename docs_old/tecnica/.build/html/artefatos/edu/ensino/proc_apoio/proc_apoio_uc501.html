Objetivo¶

Cadastrar ou remover o abono de faltas de alunos dentro de um período pré-definido.

Atores¶

Principais¶

Administrador, Secretário, Diretor Acadêmico: permite gerir os abonos de faltas de alunos.

Interessado¶

Aluno.

Pré-condições¶

Devem existir alunos cadastrados.

Pós-condições¶

As faltas do aluno no período de tempo cadastrado serão marcadas como abonadas.

Casos de Uso Impactados¶

  1. UC 503 - Manter Calendários Acadêmicos <v0.1> - O fechamento de período ignora as faltas de alunos marcadas como abonadas

  2. suap-artefatos-edu-ensino-meus-diarios-uc402 - Exibe os abonos cadastrados

  3. UC 403 - Lançar Faltas <v0.1> - Altera a marcação de falta abonada

Fluxo de Eventos¶

Fluxo Normal¶

  1. O caso de uso é iniciado acionando a opção ENSINO > Procedimentos de Apoio > Abonar Faltas

  2. O sistema exibe a lista de Abonos de Alunos (RIN1)

  3. O secretário seleciona a opção Adicionar ``Abonos de Faltas``

  4. O secretário informa os dados (RIN2) para identificar o Cadastro de Abono de Alunos

  5. Para cada Abono de Aluno que se deseja adicionar, o secretário deve informar os dados (RIN2)

  6. O secretário finaliza o caso de uso selecionando a opção Salvar

  7. O sistema exibe a mensagem M1

  8. O sistema apresenta a listagem do passo FN.2

Fluxo Alternativo¶

FA1 – Editar (FN.2 )¶

  1. O secretário aciona a opção Editar dentre um dos abonos disponíveis na listagem

  2. O sistema exibe o Abono do Aluno com os dados (RIN3) preenchidos

  3. O secretário informa novos valores para os dados (RIN2)

  4. Para cada Abono de Aluno que se deseja alterar, o secretário deve informar novos valores para os dados (RIN2)

  5. O secretário finaliza o caso de uso selecionando a opção Salvar

  6. O sistema exibe a mensagem M2.

  7. O sistema apresenta a listagem do passo FN.2

FA2 - Salvar e adicionar outro(a) (FN.5)¶

  1. O secretário aciona a opção Salvar e adicionar outro(a)

  2. O sistema exibe a mensagem M4.

  3. O caso de uso volta para o passo FN.5

FA3 - Salvar e continuar editando (FA1.4)¶

  1. O secretário aciona a opção Salvar e continuar editando

  2. O sistema exibe a mensagem M5.

  3. O caso de uso volta para o passo FA1.4

FA4 – Listar (FN.2)¶

  1. O secretário restringe a lista usando o filtro e/ou busca (RIN1)

  2. O sistema exibe uma lista que satisfaça aos critérios informados no passo anterior

FA5 - Visualizar um abono de aluno (FA4.2)¶

  1. O secretário aciona a opção Visualizar dentre um dos abonos disponíveis na listagem

  2. O sistema exibe as informações referentes ao abono de um aluno (RIN3)

FA6 - Excluir um abono de um aluno (FA5.2)¶

  1. O secretário aciona a opção Excluir disponível no início da exibição do Abonos de Faltas para excluir um determinado abono de um aluno.

  2. O sistema exibe uma tela de confirmação exibindo a mensagem M6.

  3. O secretário confirma.

  4. O sistema exclui o abono e exibe a mensagem M7.

Fluxo de Exceção¶

FE1 – Cadastro fere Regra RN2 (FN-1)¶

  1. O sistema exibe a mensagem M3.

Regras de Negócio¶

Regra

Descrição / Mensagem

RN1

O secretário e o diretor só podem criar, editar, excluir e visualizar abonos de alunos de seu próprio campus.

RN2

A Data de Fim deve ser maior que a Data de Início.

RN3

Ao cadastrar um abono, todas as faltas cadastradas do aluno entre a Data de Início e a Data de Fim tem seu estado alterado para abonada.

RN4

Ao excluir um abono, todas as faltas cadastradas do aluno entre a Data de Início e a Data de Fim tem seu estado alterado para não abonada.

RN5

Durante o caso de uso de fechamento de período só serão computadas as faltas que não tiverem o estado de abonada.

RN6

Apenas alunos cujo período letivo encontra-se aberto (matriculado) podem ter faltas abonadas ou rejeitadas.

RN7

Durante o caso de uso de Lançar Falta (UC 403 - Lançar Faltas <v0.1>), as faltas para o aluno e período especificados, serão automaticamente abonadas.

RN8

Somente alunos com período aberto podem ser acessados e com o mesmo campus do secretário ou diretor devem ser manipulados.

M8

M9

Cenário de Testes¶

Fluxo Normal¶

Objetivo

Testar se está cadastrando um abono de aluno com período aberto

Dados de Entrada

Aluno com o período aberto, Data de Início menor igual a Data de Fim e Justificativa

Resultado Esperado

  1. Uma abono de falta é cadastrado com os dados informados.

  2. Mensagem M1.

Fluxo de Exceção 01¶

Objetivo

Testar se a data de início é maior que a data de fim (RN2)

Dados de Entrada

Aluno, Data de Início maior que a Data de Fim e Justificativa

Resultado Esperado

  1. Nenhum abono de falta é cadastrado.

  2. Mensagem M3.

Regra de Negócio 01¶

Objetivo

Testar se o secretário e o diretor estão no mesmo campus do aluno (RN1)

Dados de Entrada

Aluno com período ativo de campus diferente do campus do secretário, Data de Início maior que a Data de Fim e Justificativa

Resultado Esperado

  1. Nenhum abono de falta é cadastrado.

  2. Mensagem M8.

Regra de Negócio 08¶

Objetivo

Testar se está localizando apenas alunos no período em aberto (RN8)

Dados de Entrada
Matricule os alunos João Paulo e João Marcos no período 2014.1 | Matricule João da Silva no periodo 2014.2 | Feche o período 2014.1 | Busca aluno João

Resultado Esperado

Trazer a lista com apenas João da Silva
