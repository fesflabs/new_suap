Início do Documento

Jailton Carlos

Objetivo¶

Permite o usuário gerenciar (cadastrar, alterar, remove ou listar) uma atividade complementar à prática profissional do aluno.

Atores¶

Principais¶

Secretário, diretores acadêmicos, coordenadores ou administradores do sistema.

Interessado¶

Não se aplica.

Pré-condições¶

Aluno matriculado no período.

Pós-condições¶

Atualizar matricula do aluno conforme fluxo.

A carga horária acumulada será contabilizada dentro do cumprimento da prática profissional.

Casos de Uso Impactados¶

Não há.

Fluxo de Eventos¶

Fluxo Normal¶

  1. O caso de uso é iniciado a partir do passo FA5.2 do caso de uso UC 202 - Gerir Aluno <v0.1>, ao acionar a opção ENSINO > Alunos e Professores > Alunos, em seguida, selecionando a opção Ver do aluno dentre um dos alunos disponíveis na listagem

  2. O secretário aciona a opção Atividades Complementares

  3. O sistema exibe a lista de atividades complementares (RIN1)

  4. O secretário aciona a opção Adicionar Atividade Complementar

  5. O secretário informa os dados (RIN2)

  6. O secretário finaliza o caso de uso selecionando a opção Salvar

  7. O sistema exibe a mensagem M1

  8. O sistema apresenta a listagem do passo FN.3

Fluxo Alternativo¶

FA1 - Editar (FN.2 )¶

  1. O secretário aciona a opção Editar de uma das atividades complementares disponíveis na listagem

  2. O sistema exibe a atividade complementar com os dados (RIN2) preenchidos

  3. O secretário informa novos valores para os dados (RIN2)

  4. O secretário finaliza o caso de uso selecionando a opção Salvar

  5. O sistema exibe a mensagem M2.

  6. O sistema apresenta a listagem do passo FN.3

FA2 - Visualizar (FN.2)¶

  1. O secretário aciona a opção Visualizar de uma das linhas da atividade complementar disponíveis na listagem

  2. O sistema exibe informações da registro de projeto final (RI1_)

FA3 - Remover (FA1.2)¶

  1. O secretário aciona a opção Apagar

  2. O sistema exibe a mensagem M3

  3. O secretário confirma a exclusão.

  4. O sistema apresenta a listagem do passo FN.3

Fluxo de Exceção¶

Não há.

Regras de Negócio¶

Regra

Descrição / Mensagem

RN1
Um tipo de atividade é considerada como AACC, se o tipo estiver presente em uma configuração AACC e está estiver relacionada a matriz do curso cujo aluno está matriculado.
mensagem: não há

RN2
A caixa Dados da AACC presente na Figura 2 só será exibida se a regra RN1 for cumprida.
mensagem: não há

RN3
Após qualquer alteração (Inclusão, edição, exclusão) o sistema deverá verificar se o aluno concluiu o curso, para isso, deverá ser executado o fluxograma de atualização de situação de matrícula.
Ver fluxo na Figura 3.

RN4
Critério para cálculo da carga horária no período:

  • máx. no período: é cálculada com base na “Pontuação máxima no período” multiplicado pelo “Fator de conversão (peso)”, ambos definidos na configuração de AACC da matriz do curso.

  • máx no curso: é cálculada com base na “Pontuação máxima no curso” multiplicado pelo “Fator de conversão (peso)”, ambos definidos na configuração de AACC da matriz do curso.

  • cumprida: é o somatório de todas as cargas horárias das atividades registradas do aluno cujo tipo de atividade enquadra na regra RN1

  • considerada: aplica-se o mesmo somatório da cumprida, mas considerar o limite máximo no período ao somatório.

mensagem: não há.
