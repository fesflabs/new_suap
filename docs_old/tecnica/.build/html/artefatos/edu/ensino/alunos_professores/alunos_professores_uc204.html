Objetivo¶

O usuário poderá cancelar um procedimento de trancamento ou cancelamento de matrícula.

Atores¶

Principais¶

Administrador do sistema, Diretor Acadêmico e Secretário: podem cancelar um procedimento sobre a matrícula do aluno.

Interessado¶

Aluno.

Pré-condições¶

  1. O aluno está com a matrícula cancelada (por evasão, cancelamento voluntário/compulsório, jubilamento e transferências externa/interna) ou trancada (trancamento voluntário, compulsório e intercâmbio).

Pós-condições¶

  1. Aluno fica com a situação da matrícula e período restaurados para o estado anterior.

Casos de Uso Impactados¶

  1. UC 202 - Gerir Aluno <v0.1> - Adiciona a aba de procedimentos de apoio e altera a situação do aluno e do período corrente do aluno para um estado anterior.

  2. UC 402 - Gerir Diários <v0.1> - Altera a situação do aluno no diário para um estado anterior.

  3. UC 400 - Gerir alunos em turmas <v0.3> - Altera a situação do aluno no diário para um estado anterior.

Fluxo de Eventos¶

Fluxo Normal¶

  1. O caso de uso é iniciado acionando a opção ENSINO > Alunos e Professores > Aluno

  2. O sistema exibe a lista de alunos (RIN1 – Campos para listagem de alunos)

  3. O Secretário visualiza o aluno ativo.

  4. O sistema exibe as informações sobre o aluno (RI1 – Exibição do aluno).

  5. O Secretário seleciona a opção Procedimentos de Matrícula

  6. O sistema exibe a lista de procedimentos realizados sobre a matrícula do aluno (RI1).

Fluxo Alternativo¶

FA1 - Cancelar Procedimento (FN.6 )¶

  1. O Secretário seleciona o estado atual da matrícula e seleciona a opção Excluir.

  2. O sistema exibe a mensagem de alerta M1

  3. O Secretário confirma.

  4. O sistema exibe a mensagem M2 e apresenta uma nova listagem do passo FN.6

Fluxo de Exceção¶

FE1 – Exclusão fere Regra RN2 (FA1)¶

  1. O sistema exibe a mensagem M3.

Especificação suplementares¶

Requisitos Não-Funcionais¶

Não há.

Requisitos de Interface¶

RI1 – Exibição dos procedimentos de matrícula¶

Os dados da matrícula são exibidos dentro de uma caixa de nome “Procedimentos realizados”, são eles:

  • Ano/Periodo Letivo: <campo ano>.<campo periodo_letivo>

  • Procedimento: <campo situacao_matricula>

  • Secretário: <campo servidor>

  • Data: <campo data>

  • Ação: <botão lixeira>

A Figura 1 exibe um esboço de como esses dados serão exibidos.

Requisitos de Informação¶

Não há.

Regras de Negócio¶

Regra

Descrição / Mensagem

RN1

O procedimento de cancelamento deve restaurar o estado da situação (matrícula, período, disciplinas e diários) do aluno para o estado no qual ele se encontrava anteriormente ao procedimento de cancelamento/trancamento.

RN2

Apenas o último procedimento de cancelamento/trancamento pode ser desfeito.

RN3

O botão de exclusão só aparece para o último procedimento caso sua situação seja de cancelamento/trancamento.

Mensagens¶

Código

Descrição

M1

Você tem certeza que deseja excluir o procedimento?

M2

Procedimento cancelado com sucesso.

M3

Erro ao cancelar procedimento.
