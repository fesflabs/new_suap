Objetivo¶

Obtém o tempo de serviço atual do servidor no órgão que estar lotado no momento. Este caso de uso é uma inclusão do caso de uso base chamado Exibir histórico do servidor.

Atores¶

Principais¶

Operador do RH

Interessado¶

Servidor

Pré-condições¶

  • O Servidor deve está cadastrado no sistema

  • O servidor tem que ter histórico funcional na instituição

Pós-condições¶

  • O sistema mostrará o tempo de serviço atual do servidor

Fluxo de Eventos¶

Fluxo Normal¶

  1. O caso de uso é iniciado acionando a opção Gestão de Pessoas > Mapa de tempo de serviço > Tempo de serviço atual (RI1_)

  2. O operador do RH entra com a matrícula ou nome do servidor

  3. O operador do RH finaliza o caso de uso selecionando a opção Obter tempo de serviço

  4. O sistema exibe o tempo de serviço atual do servidor

Cenário de Testes¶

Objetivos¶

O objetivo desde Caso de Testes é identificar o maior número possível de cenários e variações dos requisitos de software desde Caso de Uso. É dado um conjunto de dados de entradas, condições de execução, resultados esperados que visam validar esse caso de uso.

Casos e Registros de Teste¶

Fluxo normal¶

Objetivo

Testar a funcionalidade de obter o tempo de serviço de um regime jurídico

Dados de Entrada

Data início e data fim do regime jurídico

Resultado Esperado

Retornar um numero não negativo de dias

Fluxo normal¶

Objetivo

Testar a funcionalidade de obter o tempo de serviço ficto de um PCA

Dados de Entrada

Regimes jurídicos associados ao PCA

Resultado Esperado

Retornar um número não negativo de dias, considerando o fator de ajuste para cada regime jurídico
