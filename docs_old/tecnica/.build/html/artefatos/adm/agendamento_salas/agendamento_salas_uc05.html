Objetivo¶

Um avaliador poderá cancelar uma agenda de sala.

Atores¶

Principais¶

Avaliador: usuário autenticado pertecente ao grupo “Servidores” e que seja avaliador em ao menos uma sala.

Interessado¶

Não se aplica.

Pré-condições¶

Não há.

Pós-condições¶

  • É enviado um e-mail para o solicitante da reserva que originou a agenda.

Fluxo de Eventos¶

Fluxo Normal¶

  1. O caso de uso é iniciado acionando a opção ADMINISTRAÇÃO > Reservas de Salas > Cancelamentos

  2. O avaliador informa a sala e o período da agenda (RIN1_)

  3. O sistema exibe a lista de datas conforme agenda da sala e no período especificados (RIN2)

  4. O avaliador seleciona uma ou mais agendas da listagem e aciona a opção Excluir Selecionadas

  5. O sistema exibe a mensagem M1

  6. O avaliador confirma a exclusão

  7. O sistema apresenta a listagem do passo FN.3

Fluxo Alternativo¶

Não há.

Fluxo de Exceção¶

FE1 – Exclusão fere Regra RN1 (FN.4)¶

  1. O sistema exibe a mensagem especifica na regra de negócio RN1

Regra

Descrição / Mensagem

RN1
Listar somente as salas que possuem agenda na qual o usuário corrente é avaliador.
mensagem: não há.

RN2
Não permitir excluir uma genda com data final menor que data atual
mensagem: Impossível realizar está ação, agenda já transcorrida. Há seleção de data menor que hoje.
